## 2026-03-21

### TypeScript SDK v0.4.4 ‚Äî Enriched Data Methods

Eight new typed methods on the `PolyNode` client for all enriched data endpoints:

- `leaderboard()` ‚Äî top 20 traders by profit or volume, with period filtering
- `trending()` ‚Äî carousel, breaking markets, hot topics, featured events, and biggest movers
- `activity()` ‚Äî platform-wide trade feed (50 most recent)
- `movers()` ‚Äî markets with largest 24h price swings
- `traderProfile(wallet)` ‚Äî full trader stats: PnL, volume, trades, largest win
- `traderPnl(wallet, { period })` ‚Äî cumulative PnL time series
- `event(slug)` ‚Äî full event detail with all sub-markets
- `marketsByCategory(category)` ‚Äî browse markets by category

All methods are fully typed with dedicated response interfaces. Install:

```bash
npm install polynode-sdk@0.4.4
```

---

### TypeScript SDK v0.4.3 ‚Äî Cache UI Primitives

Four new features for building dashboards on top of the local cache.

- **View methods** ‚Äî `watchlistSummary()`, `walletDashboard()`, `leaderboard()`, `marketOverview()`. Pre-shaped data for common dashboard patterns. No SQL, no aggregation.
- **Reactive subscriptions** ‚Äî `onChange()` and `onWalletChange()` fire callbacks when new trades or settlements land from the live WebSocket stream. Returns an `unsub()` function for cleanup.
- **Export helpers** ‚Äî `exportCSV()`, `exportJSON()`, `exportRows()` dump filtered data for charting libraries, spreadsheets, or custom analysis.
- **Query builder** ‚Äî Chainable fluent API: `.wallet()`, `.side()`, `.since()`, `.market()`, `.minPnl()`, `.limit()`, `.run()`. Complex filters without writing SQL.

```bash
npm install polynode-sdk@0.4.3
```

Full documentation: [Local Cache](/sdks/local-cache)

## 2026-03-21

### API ‚Äî Enriched Data Endpoints

Eight new REST endpoints for trader analytics, market discovery, and platform trends.

- `GET /v1/leaderboard` ‚Äî Top 20 traders ranked by profit or volume (daily, weekly, monthly, all-time)
- `GET /v1/trader/{wallet}` ‚Äî Full trader profile: PnL, volume, trade count, largest win, portfolio value
- `GET /v1/trader/{wallet}/pnl` ‚Äî PnL time series at 4 resolutions (1D, 1W, 1M, ALL)
- `GET /v1/trending` ‚Äî Carousel highlights, breaking markets, hot topics, featured events, biggest movers
- `GET /v1/activity` ‚Äî Platform-wide live trade feed (last 50 trades with tx hashes)
- `GET /v1/event/{slug}` ‚Äî Full event data with all sub-markets, outcome prices, condition IDs
- `GET /v1/movers` ‚Äî Markets with the largest 24h price changes
- `GET /v1/markets/{category}` ‚Äî Category market listings with counts (crypto, politics, sports, etc.)

Rate limit: 1 request per second per API key. Responses cached 1-3 minutes.

### TypeScript SDK v0.4.1

- **Fix:** ESM imports now work correctly. v0.4.0 crashed on `cache.start()` when using `import` syntax.
- **Fix:** Eliminated `MODULE_TYPELESS_PACKAGE_JSON` Node.js warning.
- **New:** `getActiveTestWallet()` / `getActiveTestWallets()` ‚Äî returns known-active wallet addresses for testing and examples.

```bash
npm install polynode-sdk@0.4.1
```

### API ‚Äî Wallet positions fix

- Fixed `firstTradeAt` / `lastTradeAt` returning `null` for certain wallets on `GET /v1/markets/{id}/positions?includeTrades=true`.
- Added fallback query path for wallets where the primary trade lookup returns empty.

### API ‚Äî Per-endpoint rate limiting

- `includeTrades=true` on market positions now has its own 20 req/min rate limit per key.
- Separate bucket from standard rate limit ‚Äî doesn't consume your normal quota.

## 2026-03-21

### TypeScript SDK v0.4.0

- **New:** Local Cache ‚Äî SQLite-backed local storage for instant offline queries.
- Backfill wallet history in seconds (1 request per wallet, up to 500 trades).
- Live WebSocket stream keeps the cache up to date automatically.
- Query trades, positions, and settlements locally with zero API calls.
- Watchlist file with hot-reload and runtime add/remove API.
- Full documentation at [Local Cache](/sdks/local-cache).

### Rust SDK v0.4.0

- **New:** Local Cache ported from TypeScript. Same architecture, same SQL schema, same query methods.
- Builder pattern: `PolyNodeCache::builder(client).db_path(...).build()?`
- `rusqlite` (bundled) + `notify` as optional dependencies behind `cache` feature flag.

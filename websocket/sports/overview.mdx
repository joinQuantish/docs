## Endpoint

```
wss://sports.polynode.dev/ws?key=pn_live_YOUR_KEY
```

Authentication is the same `pn_live_*` API key used for the rest of polynode. Connection limits and rate limits follow your tier.

## Quick start

<Steps>
  <Step title="Connect">
    <CodeGroup>
    ```javascript Node.js
    import { WebSocket } from "ws";

    const ws = new WebSocket("wss://sports.polynode.dev/ws?key=pn_live_YOUR_KEY");

    ws.on("open", () => {
      ws.send(JSON.stringify({
        action: "subscribe",
        type: "sports_scores"
      }));
    });

    ws.on("message", (raw) => {
      const msg = JSON.parse(raw.toString());
      if (msg.type === "score_update") {
        const g = msg.data;
        console.log(`${g.awayTeam} ${g.score} @ ${g.homeTeam} (${g.period}) — ${g.leagueAbbreviation}`);
      }
    });
    ```

    ```python Python
    import asyncio, json
    import websockets

    async def stream():
        url = "wss://sports.polynode.dev/ws?key=pn_live_YOUR_KEY"
        async with websockets.connect(url) as ws:
            await ws.send(json.dumps({
                "action": "subscribe",
                "type": "sports_scores"
            }))
            async for raw in ws:
                msg = json.loads(raw)
                if msg.get("type") == "score_update":
                    g = msg["data"]
                    print(f"{g['awayTeam']} {g['score']} @ {g['homeTeam']} ({g['period']}) — {g['leagueAbbreviation']}")

    asyncio.run(stream())
    ```
    </CodeGroup>
  </Step>

  <Step title="Receive a snapshot of every live game">
    Immediately after subscribing you'll receive the current state of every live game that matches your filters. Each event arrives as a `score_update` message. A `snapshot_complete` message follows once the backlog is fully delivered.
  </Step>

  <Step title="Continue receiving updates as games progress">
    After the snapshot, every score change, period change, or game state transition is pushed to your connection in real time.
  </Step>
</Steps>

## Coverage

The Sports WebSocket covers every sport with a live game in our system. Active categories include:

- **Tennis** — ATP, WTA, Challenger, ITF
- **Basketball** — NBA, WNBA, NCAA, EuroLeague, FIBA, China CBA, Korean KBL, and more
- **Soccer** — Premier League, La Liga, Serie A, Bundesliga, Ligue 1, MLS, Brazilian Série A, Champions League, World Cup
- **American Football** — NFL, NCAA
- **Hockey** — NHL, AHL, KHL, SHL, DEL
- **Baseball** — MLB, KBO, NPB
- **Cricket** — IPL, BBL, PSL, T20 Internationals
- **MMA** — UFC
- **Esports** — League of Legends, Counter-Strike, Dota 2, Valorant, Mobile Legends, Rainbow Six, Call of Duty
- **Other** — Chess, lacrosse, table tennis, motorsport

Use the [REST `/games` snapshot](/websocket/sports/games-rest) to see exactly what's currently live.

## Message types you'll receive

| Type | Sent when |
|---|---|
| `subscribed` | Acknowledges your subscribe request, echoes the resolved filters |
| `score_update` | A live game has new state (the main payload — see [event format](/websocket/sports/event-format)) |
| `snapshot_complete` | Backlog replay is done after a fresh subscribe |
| `unsubscribed` | Acknowledges your unsubscribe |
| `heartbeat` | Sent every 30 seconds — used for connection liveness |
| `pong` | Response to a `{"action":"ping"}` sent by your client |
| `error` | Sent if something is wrong (invalid filter, etc.) |

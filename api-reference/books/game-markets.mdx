Returns the complete set of market snapshots we have for one game. Same data as the `markets` array from [Game Detail](/api-reference/books/game-detail), but flattened to just the odds — useful when you already have the game metadata cached client-side.

```bash
curl -s "https://books.polynode.dev/v1/games/40664-16839-2026-04-14/markets?key=pn_live_YOUR_KEY"
```

<ParamField path="id" type="string" required>
  Polynode slug or raw `game_id`.
</ParamField>

<ParamField query="key" type="string" required>
  Your API key.
</ParamField>

## Top-level shape

| Field | Type | Description |
|---|---|---|
| `game_id` | string | Upstream canonical id. |
| `pn_slug` | string \| null | Polynode slug if resolvable. |
| `count` | integer | Number of market snapshots returned. |
| `markets` | array | Market snapshot objects — same shape as in [Game Detail](/api-reference/books/game-detail#market-snapshot-object). |

The exact set varies by sport and whether the game is pre-match, live, or post-final. NBA games typically carry moneyline, spreads, totals, plus player_points, player_rebounds, player_assists, player_made_threes, player_double_double, and the 1st-half variants. Football carries player yard props and anytime TD. Soccer adds BTS, correct score, corners, anytime goalscorer, and halftime result. Baseball adds nrfi. Tennis uses set handicap and total games. UFC uses method of victory and go the distance.

## Interpreting odds that move with the score

In the example above, Cusco FC moneyline is priced at **+5000 to +6600** across multiple books. That's because they are losing **2-1** in the 50th minute of the Copa Libertadores game shown. Moneyline odds move sharply against the trailing team in live markets — which is exactly the sort of signal this endpoint surfaces in near-real-time.

## Refresh semantics

- Live games are re-polled every ~1 second, so in-play markets reflect upstream within 1-2 seconds.
- Idle games (unplayed or final) are re-polled every ~10 seconds.
- `content_hash` changes only when a price, point, or outcome set actually mutates — safe to use for client-side change detection to avoid re-rendering.

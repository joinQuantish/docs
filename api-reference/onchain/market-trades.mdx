---
title: "Trade History (Market)"
description: "Complete onchain trade history for any market token."
openapi: "GET /v2/onchain/markets/{token_id}/trades"
---

Returns every onchain trade fill for a specific market token. This is the complete trade tape from the blockchain, enriched with market metadata.

<Tip>
**Need to bulk-export the entire market in one call?** See [Trade History (Market) — Bulk](/api-reference/onchain/market-trades-all) — Growth plan and above can fetch the full trade history in one response (no pagination, up to 250K trades).
</Tip>

## Two pagination modes

This endpoint supports both **offset** and **cursor** pagination. They produce identical data; cursor is much faster at depth.

| Mode | Trigger | Speed | When to use |
|---|---|---|---|
| **Offset** (default) | `?offset=N` (or no pagination params) | Slows linearly with depth — fast for the first ~25K results, then degrades | Casual use, small markets, getting just the first page |
| **Cursor** (recommended for paging) | Pass `?cursor=` (empty for first page, then `?cursor=<lastTs>:<lastId>` from prior response) | **Constant ~1 second per page at any depth** | Walking the entire trade history of a market, deep pagination, large markets |

**Cursor walks any market in seconds-per-page regardless of how many trades it has.** Available on every paid tier — no growth restriction.

For a one-shot bulk export of an entire market in a single response (no pagination needed), see [Trade History (Market) — Bulk](/api-reference/onchain/market-trades-all) — Growth plan and above.

## Request

```
GET /v2/onchain/markets/{token_id}/trades?limit=100&offset=0
GET /v2/onchain/markets/{token_id}/trades?limit=1000&cursor=<cursor>
```

| Parameter | Type | Location | Description |
|-----------|------|----------|-------------|
| `token_id` | string | path | CTF token ID (numeric string) |
| `limit` | integer | query | Max results (default 100, max 1000) |
| `offset` | integer | query | Offset mode: skip first N results. Used only when `cursor` is absent. Slow at depth (>25K). |
| `cursor` | string | query | Cursor mode: pass empty string `?cursor=` for first page, then echo back `pagination.cursor` from the response for each subsequent page. Format: `<lastTs>:<lastId>`. Constant-time at any depth. |

## Example: simple offset (first page)

```bash
curl "https://api.polynode.dev/v2/onchain/markets/110959653450933276250915064669875552310439627880508793089816880777942697720191/trades?limit=20" \
  -H "x-api-key: YOUR_KEY"
```

## Example: cursor pagination — walk an entire market

For markets with thousands of trades, use cursor mode. Each page is constant-time (~1 second) regardless of depth.

**Page 1 (first page, no cursor yet — pass empty `?cursor=`):**

```bash
TOK=110959653450933276250915064669875552310439627880508793089816880777942697720191
curl "https://api.polynode.dev/v2/onchain/markets/$TOK/trades?limit=1000&cursor=" \
  -H "x-api-key: YOUR_KEY"
```

**Page 2+ — echo back the `cursor` from the previous response:**

```bash
curl "https://api.polynode.dev/v2/onchain/markets/$TOK/trades?limit=1000&cursor=1777595648:0xeae6ca0aeb75fc866ddb29d3a4b2cd15382895b923fe05b9315445d6346d1c47_0x49ddd1e169186d350488e3d62f58452fe76862c5f81f33987e659759e2c66100" \
  -H "x-api-key: YOUR_KEY"
```

Repeat until `pagination.has_more` is `false`. Done — you've walked the entire trade history of the market.

**Cursor format:** `<unix_timestamp>:<txHash_orderHash>`. Treat it as opaque — pass it back unchanged. Server-side it filters trades where `(timestamp < lastTs) OR (timestamp == lastTs AND id < lastId)`, so same-timestamp ties (multiple fills in one block) are handled correctly with no dups or skips.

## When to use which

- **First page only / interactive UI** → offset mode, no params (`?limit=100`).
- **Walking the full history** → cursor mode (`?cursor=` then echo back).
- **One-shot bulk export of the entire market** → [`/trades/all`](/api-reference/onchain/market-trades-all) (Growth plan).

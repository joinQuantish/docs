## What is PolyNode RPC?

PolyNode exposes a standard JSON-RPC 2.0 interface for the Polygon network. You can use it as a drop-in replacement for any Polygon RPC provider — with the bonus of custom `pn_*` methods for Polymarket-specific data.

## Endpoint

```
POST https://api.polynode.dev/rpc
```

All RPC calls go through a single endpoint as POST requests with a JSON-RPC 2.0 body.

## Standard vs Custom Methods

| Prefix | Source | Description |
|--------|--------|-------------|
| `eth_*` | Polygon network | Standard Ethereum JSON-RPC methods with intelligent caching |
| `net_*` | Polygon network | Network info methods |
| `txpool_*` | Polygon network | Transaction pool methods |
| `pn_*` | PolyNode | Custom Polymarket methods with sub-millisecond response times |

## Authentication

RPC requests require an API key, same as all other authenticated endpoints. Pass your key via the `x-api-key` header.

## Request Format

```json
{
  "jsonrpc": "2.0",
  "method": "eth_blockNumber",
  "params": [],
  "id": 1
}
```

## Batch Requests

Send an array of requests to process up to 100 calls in a single HTTP request:

```json
[
  {"jsonrpc": "2.0", "method": "eth_blockNumber", "params": [], "id": 1},
  {"jsonrpc": "2.0", "method": "eth_gasPrice", "params": [], "id": 2}
]
```

Returns an array of responses in matching order. Maximum 100 requests per batch.

## Caching

PolyNode applies intelligent caching for optimal performance:

| Cache Tier | TTL | Methods |
|------------|-----|---------|
| Passthrough | 0s | `eth_sendRawTransaction`, `eth_call`, `eth_estimateGas`, `eth_getTransactionCount` |
| Short | 2s | `eth_blockNumber`, `eth_gasPrice`, `net_peerCount`, `eth_getBalance`, `eth_getStorageAt`, `eth_feeHistory` |
| Medium | 30s | `eth_getBlockByNumber`, `eth_getBlockByHash`, `eth_getLogs`, `txpool_status` |
| Permanent | 24h | `eth_getTransactionReceipt`, `eth_getTransactionByHash`, `eth_chainId`, `net_version`, `eth_getCode` |
| Local | instant | `web3_clientVersion` (returns `"PolyNode/1.0.0/rust"`), all `pn_*` methods |

Custom `pn_*` methods return live data with sub-millisecond response times.

## Historical Data

PolyNode maintains recent block data. For deep historical queries (blocks older than ~24 hours), pair with an archive node provider.

## Error Codes

Standard JSON-RPC 2.0 error codes:

| Code | Message | Description |
|------|---------|-------------|
| -32700 | Parse error | Invalid JSON |
| -32600 | Invalid Request | Missing required fields or empty batch |
| -32601 | Method not found | Unknown method name |
| -32603 | Internal error | Node unavailable or processing failure |

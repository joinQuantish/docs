---
title: "Global Trades"
openapi: "GET /v3/trades"
description: "Browse the full Polymarket trade history with market enrichment, computed prices, and order hashes."
---

Returns all Polymarket fills across all wallets, enriched with market metadata. Supports time range filtering, dollar-size filtering, and scoping by token, market, or event.

## Request

```
GET /v3/trades
```

### Query parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `after` | integer | -- | Start of time range (Unix timestamp) |
| `before` | integer | -- | End of time range (Unix timestamp) |
| `token_id` | string | -- | Filter by outcome token |
| `condition_id` | string | -- | Filter by market condition ID (resolves to token IDs) |
| `market_slug` | string | -- | Filter by market slug (resolves to token IDs) |
| `event_slug` | string | -- | Filter by event slug (resolves to token IDs across all of the event's markets) |
| `event_id` | integer | -- | Filter by numeric event ID (same scope as `event_slug`) |
| `order_hash` | string | -- | Return only fills for one exact order hash (`0x` + 64 hex characters) |
| `builder` | string | -- | Filter by builder code (hex) |
| `min_amount_usd` | number | -- | Minimum trade size in dollars, matching the `amount_usd` response field. Decimals allowed, e.g. `99.5` |
| `min_amount` | integer | -- | Minimum maker_amount_filled (raw 6-decimal). Prefer `min_amount_usd` |
| `side` | string | -- | Maker order side: `buy`, `sell`, `0`, or `1` |
| `sort_by` | string | -- | `order_hash` groups fills from the same order visually |
| `group_by` | string | -- | `order_hash` aggregates the returned fills per order |
| `order` | string | `desc` | `asc` or `desc` |
| `limit` | integer | 100 | Max 300 |
| `offset` | integer | 0 | Pagination offset |

Event filters cover events currently in Polymarket's catalog. A very large event can return `400` asking for a narrower filter (`condition_id` or `market_slug`).

All supplied filters are intersected. `order_hash` is an exact, indexed filter
applied before pagination. Hex characters are case-insensitive; malformed or
empty hashes return `400`, while a valid hash with no fills returns an empty
`200` page.

## Example

Every fill for one order:

```bash
curl -H "x-api-key: pn_live_YOUR_KEY" \
  "https://api.polynode.dev/v3/trades?order_hash=0x9ed363e9c05368fba5c717b8bee355d60a579f599cb7c007d90974203896a95c"
```

See [Trades by Order Hash](/data/query-catalog/trades/by-order-hash) for global,
wallet, token, slug, pagination, and grouping recipes.

$100+ trades, newest first:

```bash
curl -H "x-api-key: pn_live_YOUR_KEY" "https://api.polynode.dev/v3/trades?min_amount_usd=100&limit=1"
```

Event-scoped: every trade across all markets of one event:

```bash
curl -H "x-api-key: pn_live_YOUR_KEY" "https://api.polynode.dev/v3/trades?event_slug=fifwc-arg-che-2026-07-11&limit=100"
```

## Response fields

Each trade contains the same fields as [Wallet Trades](/data/wallets/trades), including market enrichment (market, slug, outcome, image, condition_id, event_slug, event_id), computed price and size in USD, order_hash, and direction. `event_slug`/`event_id` identify the Polymarket event the market belongs to; they are null for old markets that no longer exist in Polymarket's catalog.

On global trades (no wallet context), `role` is always `"maker"` and `direction` reflects the maker's action. Use the [Wallet Trades](/data/wallets/trades) endpoint for wallet-specific perspective.

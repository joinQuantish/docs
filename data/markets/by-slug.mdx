---
title: "Market by Slug"
description: "Look up trades and positions for a market using its human-readable URL slug."
---

Look up market data using the slug from Polymarket URLs. Resolves the slug to token IDs internally and returns trades or positions across all outcomes.

## Trades by slug

```
GET /v3/markets/slug/{slug}/trades
```

### Query parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `after` | integer | — | Start of time range (Unix timestamp) |
| `before` | integer | — | End of time range |
| `limit` | integer | 100 | Max 1000 |
| `offset` | integer | 0 | Pagination offset |

## Positions by slug

```
GET /v3/markets/slug/{slug}/positions
```

### Query parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `sort` | string | `pnl` | Sort by: `pnl`, `amount`, `volume` |
| `limit` | integer | 100 | Max 1000 |
| `offset` | integer | 0 | Pagination offset |

## Example

```bash
curl https://api.polynode.dev/v3/markets/slug/bitcoin-above-82k-on-january-16/trades?limit=5
```

Response includes standard fill columns: `id`, `maker`, `taker`, `maker_asset_id`, `taker_asset_id`, `maker_amount_filled`, `taker_amount_filled`, `fee`, `timestamp`, `transaction_hash`, `builder`, `side`.

Returns 404 if the slug doesn't match any market in the metadata table.

---
title: "Tag Leaderboard"
openapi: "GET /v3/tags/{slug}/leaderboard"
description: "Rank wallets by P&L within a specific market category (tag). Works for niche tags."
---

Returns a P&L leaderboard filtered to markets with a specific tag. Works for smaller/niche tags. Large tags (sports, crypto) return an error suggesting the global leaderboard instead.

## Request

```
GET /v3/tags/{slug}/leaderboard
```

### Query parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `sort` | string | `net_pnl` | Sort by `net_pnl` (default), `volume`, `profit`, or `loss` |
| `limit` | integer | 100 | Max 1000 |
| `offset` | integer | 0 | Pagination offset |

### Slug handling

The tag slug is matched case-insensitively. URL-style hyphens are also converted to spaces, so `/v3/tags/us-election/leaderboard` resolves to the stored tag `US Election`.

### Tag size cap

Tags with more than 4,000 markets (`Sports`, `Crypto`, `Politics`, etc.) return a structured error instead of running the real-time aggregation:

```json
{
  "error": "tag too large for real-time leaderboard",
  "tag": "sports",
  "market_count": 575318,
  "hint": "use /v3/leaderboard for global rankings, or filter by a more specific tag",
  "elapsed_ms": 1
}
```

Use [`GET /v3/leaderboard`](/data/leaderboard/global) for global rankings or pick a more specific tag.

## Example

```bash
curl https://api.polynode.dev/v3/tags/taylor-swift/leaderboard?limit=1
```

## Response fields (per row in `data`)

| Field | Type | Description |
|--------|------|-------------|
| `wallet` | string | Wallet address |
| `net_pnl` | string (decimal) | Net realized P&L in this tag (USD) |
| `gross_profit` | string (decimal) | Sum of winning positions (USD) |
| `gross_loss` | string (decimal) | Sum of losing positions (USD, negative or zero) |
| `total_volume` | string (decimal) | Total volume traded in this tag (USD) |
| `position_count` | integer | Number of positions in markets with this tag |

---
title: "position_status_change"
description: "Fires when a position transitions between states: open, closed, redeemable, or redeemed."
---

<Note>**polynode exclusive** — no other Polymarket data provider tracks position lifecycle state transitions.</Note>

## Filters

| Filter | Type | Description |
|---|---|---|
| `wallet_addresses` | `string[]` | Wallets to watch. Max 500. |
| `status_from` | `string` | Previous status (`open`, `closed`, `redeemable`, `redeemed`). |
| `status_to` | `string` | New status. |

## States

| Status | Meaning |
|---|---|
| `open` | Wallet holds shares in an active market |
| `closed` | Position fully sold before resolution |
| `redeemable` | Market resolved, shares can be redeemed |
| `redeemed` | Shares redeemed, USDC claimed |

## Use Cases

- Settlement bots: trigger auto-redemption when positions become redeemable
- Portfolio trackers: show status transitions in real-time
- Copy-trading: detect when a leader fully exits a position (open → closed)

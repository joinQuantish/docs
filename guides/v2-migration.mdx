Polymarket is deploying a V2 exchange system on Polygon mainnet. polynode detected, decoded, and verified every V2 contract on day one. Our settlement stream, event pipeline, and trading SDK support V2 alongside V1 with zero disruption.

**No action is required from existing settlement stream users.** V2 trades flow through the same WebSocket feed automatically.

## What We've Verified

All of the following has been tested against live V2 contracts on Polygon mainnet:

- V2 settlement calldata decoded and verified against real on-chain transactions
- V2 OrderFilled and OrdersMatched events decoded from real block receipts
- PolyUSD wrapping and unwrapping executed and detected through our P2P pipeline
- V2 order placement tested live on the V2 CLOB
- Full order lifecycle verified: place, check status, cancel, re-place
- All existing market data, token IDs, and enrichment confirmed identical between V1 and V2

## What's New in V2

### PolyUSD Collateral

V2 uses **PolyUSD** instead of USDC.e as the exchange collateral. PolyUSD is a 1:1 wrapper around USDC.e with 6 decimals. See the [PolyUSD Guide](/guides/polyusd) for details on wrapping and unwrapping.

### Updated Order Struct

The V2 order has a different set of fields than V1. Four fields were removed (`taker`, `expiration`, `nonce`, `feeRateBps`) and three were added (`timestamp`, `metadata`, `builder`). The polynode SDK handles this automatically — you don't need to construct orders manually.

### New Exchange Contracts

V2 uses three new exchange contracts for standard and multi-outcome (neg-risk) markets. The underlying ConditionalTokens contract is unchanged.

### EIP-712 Domain Update

The signing domain version changed from `"1"` to `"2"`. The SDK handles this based on your exchange version config.

## What Didn't Change

- **Token IDs** — same ConditionalTokens contract, same position IDs, same enrichment
- **Market data** — same markets, same condition IDs, same CLOB API paths
- **Authentication** — same API key derivation, same HMAC headers
- **WebSocket protocol** — same subscriptions, same event types, same format
- **Position splits/merges** — same ConditionalTokens events
- **Oracle system** — unchanged UMA adapters
- **Orderbook streaming** — same WebSocket protocol and message format

## Impact on polynode Users

### Settlement Stream

No changes required. V2 settlements produce the same event types:

- `settlement` — pending detection from mempool (3-5 seconds pre-confirmation)
- `status_update` — block confirmation with latency measurement
- `trade` — confirmed trade from OrderFilled event
- `deposit` — now includes PolyUSD wrapping/unwrapping alongside USDC.e

V2 detection is automatic. polynode identifies V2 transactions and decodes them alongside V1.

### Trading SDK

The polynode SDK supports V2 with a single config change:

<CodeGroup>
```rust Rust
use polynode::trading::{PolyNodeTrader, TraderConfig, OrderParams, OrderSide, ExchangeVersion};

let mut config = TraderConfig {
    polynode_key: "pn_live_...".into(),
    exchange_version: ExchangeVersion::V2,  // ← only change needed
    ..Default::default()
};

let mut trader = PolyNodeTrader::new(config)?;
let signer = PrivateKeySigner::from_hex("0x...")?;
let status = trader.ensure_ready(Box::new(signer), None).await?;

// Order placement is identical to V1
let result = trader.order(OrderParams {
    token_id: "10293...".into(),
    side: OrderSide::Buy,
    price: 0.05,
    size: 10.0,
    ..Default::default()
}).await?;
```

```typescript TypeScript (SDK)
// npm install polynode-sdk
import { PolyNodeTrader } from "polynode-sdk";

const trader = new PolyNodeTrader({
  polynodeKey: "pn_live_...",
  exchangeVersion: "v2",  // ← only change needed
});

await trader.ensureReady("0xprivatekey...");

// Order placement is identical to V1
const result = await trader.order({
  tokenId: "10293...",
  side: "BUY",
  price: 0.05,
  size: 10,
});
```


<Info>
The SDK handles all V2 differences internally: order struct, EIP-712 signing, CLOB endpoint routing, and PolyUSD collateral. You don't need to change how you construct orders, fund wallets, or manage positions.
</Info>

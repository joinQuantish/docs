Connect via WebSocket for real-time Polymarket events. Supports filtered subscriptions with multiple event types, including pre-chain pending transactions.

## Connecting

```
wss://ws.polynode.dev/ws?key=pn_live_YOUR_KEY
```

The API key must be passed as a query parameter.

PolyNode WebSocket supports filtered subscriptions for real-time Polymarket events.

## Subscribing

After connecting, send a JSON subscribe message:

```json
{
  "action": "subscribe",
  "type": "settlements",
  "filters": {
    "tokens": ["21742633..."],
    "side": "BUY",
    "min_size": 100,
    "status": "pending"
  }
}
```

You'll receive an acknowledgment followed by a **snapshot** of recent matching events, then live updates.

## Server heartbeat

The server sends a heartbeat every 30 seconds:
- A WebSocket-level **Ping** frame (handled automatically by all WS clients)
- A text message: `{"type": "heartbeat", "ts": 1772386305181}`

If you don't receive a heartbeat within ~35 seconds, the connection is likely dead — reconnect.

## Subscription types

| Type | Events | Description |
|------|--------|-------------|
| `settlements` | settlement, status_update | Pending + confirmed settlements (default) |
| `trades` | settlement, trade, status_update | All trade activity |
| `prices` | settlement, trade | Price-moving events for specific markets |
| `blocks` | block | New block notifications |
| `wallets` | settlement, trade, position_change, deposit, status_update | All activity for specific wallets |
| `markets` | settlement, trade, position_change, status_update | All activity for specific markets |
| `large_trades` | settlement, trade | Whale alerts ($1K+ default min_size) |
| `global` | all | Unfiltered stream of everything |

## Filters

All filters are optional. Omit filters to receive all events of the subscription type.

| Filter | Type | Description |
|--------|------|-------------|
| `wallets` | string[] | Filter by wallet addresses |
| `tokens` | string[] | Filter by token IDs |
| `markets` | string[] | Alias for `tokens` |
| `slugs` | string[] | Filter by market slugs (e.g. `["bitcoin-100k-2026"]`). Resolved to token IDs at subscribe time. |
| `condition_ids` | string[] | Filter by Polymarket condition IDs. Resolved to token IDs at subscribe time. |
| `side` | string | `"BUY"` or `"SELL"` |
| `min_size` | number | Minimum trade size in USD |
| `max_size` | number | Maximum trade size in USD |
| `status` | string | `"pending"`, `"confirmed"`, or `"all"` (default) |
| `event_types` | string[] | Override default event types |
| `snapshot_count` | number | Number of events in the initial snapshot (default 20, max 200) |
| `feeds` | string[] | Filter by price feed names (e.g. `["BTC/USD"]`). Only for `chainlink` subscriptions. |

<Tip>
Use `slugs` or `condition_ids` instead of raw `tokens` when you know the market by its URL slug or condition ID. The server resolves them to token IDs once at subscribe time — no performance cost after that.
</Tip>

## Event types

All events include a top-level **`timestamp`** field (Unix ms) — use this instead of switching on `detected_at` / `timestamp` / `confirmed_at` per event type.

All trade-related events include **enrichment fields** (`market_title`, `outcome`, `market_slug`, `event_title`, `market_image`) resolved from metadata.

## Unsubscribing

Remove a specific subscription by its ID (returned in the `subscribed` response):

```json
{"action": "unsubscribe", "subscription_id": "550e8400-...:1"}
```

Or remove all subscriptions on the connection:

```json
{"action": "unsubscribe"}
```

## Keepalive

The server sends automatic heartbeats every 30 seconds (see above). You can also send client-side pings:

```json
{"action": "ping"}
```

Response: `{"type": "pong"}`

## Error messages

Invalid messages return structured errors:

```json
{"type": "error", "code": "invalid_json", "message": "Message is not valid JSON."}
```

```json
{
  "type": "error",
  "code": "invalid_message",
  "message": "Expected: {\"action\": \"subscribe\"|\"unsubscribe\"|\"ping\", ...}"
}
```

## Examples

### Track whale trades

```javascript
const ws = new WebSocket("wss://ws.polynode.dev/ws?key=pn_live_YOUR_KEY");

ws.onopen = () => {
  ws.send(JSON.stringify({
    action: "subscribe",
    type: "large_trades",
    filters: { min_size: 5000 },
  }));
};

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  if (msg.type === "settlement") {
    console.log(
      `Whale: ${msg.data.taker_side} $${msg.data.taker_size} @ ${msg.data.taker_price}`,
      `on "${msg.data.market_title}" (${msg.data.outcome})`
    );
  }
};
```

### Monitor a specific market (by slug)

```javascript
ws.send(JSON.stringify({
  action: "subscribe",
  type: "markets",
  filters: {
    slugs: ["bitcoin-100k-2026"],
    status: "pending",  // Only pre-chain events
  },
}));
```

### Monitor by token ID

```javascript
ws.send(JSON.stringify({
  action: "subscribe",
  type: "markets",
  filters: {
    tokens: ["21742633..."],
    snapshot_count: 50,  // Get last 50 events on connect
  },
}));
```

### Track a wallet

```javascript
ws.send(JSON.stringify({
  action: "subscribe",
  type: "wallets",
  filters: { wallets: ["0xabcdef..."] },
}));
// Receives: settlements, trades, position changes, deposits, status updates
```

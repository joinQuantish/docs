---
title: "Channels"
description: "Every perps WebSocket channel and its payload."
---

| Channel | Scope | Payload |
|---------|-------|---------|
| `perps_tickers` | all instruments | Ticker updates: index/mark/last/mid price, open interest, funding rate, next funding |
| `perps_statistics` | all instruments | Rolling 24h stats per instrument, sent when they change |
| `perps_bbo:<instrument>` | one instrument | Best bid/offer changes |
| `perps_book:<instrument>` | one instrument | Complete top-20 snapshot on subscribe **and on every update** — replace your local book, never merge |
| `perps_trades:<instrument>` | one instrument | Executions as they print |
| `perps_klines:<instrument>:<interval>` | one instrument | The current candle, sent whenever it changes; intervals `1m` and `1h` |

## Example events

`perps_tickers`:

```json
{
  "channel": "perps_tickers",
  "data": {
    "instrument_id": 6,
    "symbol": "BTC-USD",
    "index_price": "62022",
    "mark_price": "62010",
    "last_price": "61989",
    "mid_price": "62010",
    "open_interest": "67.44542",
    "funding_rate": "0.0000125",
    "next_funding": 1783573200000,
    "timestamp": 1783571023159
  }
}
```

`perps_trades:<instrument>`:

```json
{
  "channel": "perps_trades",
  "data": {
    "instrument_id": 6,
    "symbol": "BTC-USD",
    "data": [
      {
        "trade_id": 7504966405671651,
        "instrument_id": 6,
        "symbol": "BTC-USD",
        "side": "short",
        "price": "62009",
        "quantity": "0.00842",
        "timestamp": 1783571031288
      }
    ]
  }
}
```

`perps_book:<instrument>` — **every message is a complete snapshot** of the top 20 levels per side, including the one you get immediately on subscribe. Replace your local book with each message; merging messages as if they were deltas will accumulate stale levels and cross the book:

```json
{
  "channel": "perps_book",
  "data": {
    "instrument_id": 6,
    "symbol": "BTC-USD",
    "bids": [["62009", "0.03999"]],
    "asks": [["62010", "0.05098"]],
    "timestamp": 1783571023159
  }
}
```

`perps_klines:<instrument>:<interval>`:

```json
{
  "channel": "perps_klines",
  "data": {
    "instrument_id": 6,
    "symbol": "BTC-USD",
    "interval": "1m",
    "candle": [1783571040000, "62010", "62015", "62001", "62009", "1.2043", 18],
    "timestamp": 1783571100000
  }
}
```

Candles are `[open_time, open, high, low, close, volume, trade_count]` — the same layout as [REST klines](/perps/market-data/klines).

Each event carries the **current** candle for the interval, sent only when its contents change (a trade prints, or a new period opens). A candle is final once you see an event with a later `open_time` for the same instrument and interval. Quiet markets send nothing — silence means no change, not a dropped connection (the 30-second server pings cover liveness).
